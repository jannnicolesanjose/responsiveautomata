Introduction

This project was developed by Aivee Varias, Jann Nicole San Jose, Timothy Sicad, and Xian Palomares as part of the requirements for the course Automata Theory and Formal Language. It presents a Deterministic Finite Automaton (DFA) solver, designed to analyze and determine whether given strings are accepted based on predefined state transitions and acceptance criteria.

The DFA solver serves as a computational model that systematically processes input strings and evaluates their validity within a structured framework. By implementing deterministic state transitions, the solver ensures precise and unambiguous results, making it a valuable tool for understanding foundational concepts in formal language theory.

This project not only demonstrates the practical application of DFA principles but also highlights the significance of automata in various computational and real-world scenarios, such as lexical analysis in programming languages and pattern recognition in data processing. Through this endeavor, the team aims to enhance comprehension of automata theory while providing an interactive approach to exploring finite state machines.

Deterministic Finite Automata (DFA)

According to Sisense (2025), Deterministic Finite Automata (DFA) are finite state machines that accept or reject strings of characters by parsing them through a sequence that is uniquely determined by each string. In DFA, for each input symbol, one can determine the state to which the machine will move. The transition from a state is to a single particular next state for each input symbol. This means that each string and each state sequence is unique. Hence, it is called deterministic.

DFA is represented by state diagram which is made up of five components:

  • A finite number of states, represented by vertices
  • A set of symbols known as the alphabet, also finite in number
  • The arcs labeled with an input alphabet show the transitions
  • The initial state where the first input is given or processed, denoted by minus/negative sign (-)
  • The final state or states, known as accepting states, denoted by plus/positive sign (+)

In this project, the DFA is converted based on the given regular expression. The string of symbols is parsed through the DFA automata, and each input symbol will move to the next state that can be determined. A string is accepted by a DFA if the DFA starting at the initial state ends in any of the final states after reading the whole string. This DFA will be used to determine and simulate whether a user’s input is valid or invalid.

User's Manual

Welcome to the Regular Expression Converter Web Application User’s Manual. This guide is designed to help users navigate and utilize the features of our web-based application developed for converting Regular Expressions into their equivalent Deterministic Finite Automata (DFA), Context-Free Grammar (CFG), and Pushdown Automata (PDA) representations.

This application was created to support students and learners in better understanding formal language theory and automata concepts by providing an interactive platform for simulation and visualization. Whether you're a beginner exploring the basics of regular expressions or a student needing visual aids for DFA, CFG, or PDA, this manual will walk you through every component of the system.

By following this manual, users will gain insights into how to:

  • Navigate different pages such as Home, ABout, Tutorial, and Documentation
  • Use the simulation features for two types of regular expression (a&b or 0&1)
  • Interpret DFA transitions with animated and visual feedback
  • View corresponding CFG and PDA of the given regular expressions
  • And efficiently clear or reset inputs

This manual ensures that the users can make full use of the system with ease and confidence by providing step-by-step guidance, clear explanations, and detailed descriptions of each feature available in the web application.
